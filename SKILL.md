# 城市级报告质检Skill v1.1

## 一、Skill目的

**避免重蹈v1.0/v2.0/v3.0报告的7个硬伤**：

| # | 硬伤 | 教训 |
|---|------|------|
| 1 | 定位模糊（既不像城市级也不像商圈级） | 报告开头必须写明"本报告定位=城市级" |
| 2 | 结论主观断言，缺数据支撑 | 每个结论必须带"数据依据"列 |
| 3 | 城市级核心指标全部缺失 | 强制8项指标清单，缺一不可交付 |
| 4 | 竞品榜单单维度（只看评价数） | 必须4维度：门店数/评价数/存活时间/区域分布 |
| 5 | 消费力+成本维度缺失 | 必须有算账表：投入/营收/回本周 |
| 6 | Checklist太泛（"临街旺铺"） | 每条带量化标准：半径/客单价±20% |
| 7 | 缺一页纸决策结论 | 第1页必须是7要素结论表 |

## 二、数据采集硬规则（v3.0经验）

### 规则1：广义品类覆盖（覆盖率≥80%）

**禁止只用字面关键词**（重蹈v2.0的1264家 vs 真实6000家）：

```
错误：只搜"火锅" → 1264家（覆盖率18%）
正确：搜24个广义关键词 → 3013家（覆盖率50%）
```

**检查点**：采集完成后计算覆盖率，<30%必须重采。

### 规则2：多源交叉验证（≥3个数据源）

**禁止单平台数据**：

```
高德POI（主数据源）
美团官方指数（标准口径）
大众点评（交叉验证）
工商注册（上限估算）
```

**v1→v3进化**：
- v1.0：15家（0.3%覆盖）❌
- v2.0：1264家（24%覆盖）❌
- v3.0：3013家（50%覆盖，24个关键词）✅

### 规则3：8项必含指标（缺一不可）

| # | 指标 | 计算方法 | 数据来源 |
|---|------|----------|----------|
| 1 | 全市门店总量 | 多源对比 | 高德+美团+点评 |
| 2 | 每万人门店数（饱和度） | 总量/常住人口 | 官方人口数据 |
| 3 | 各区门店密度 | 各区门店/区面积 | 区域面积数据 |
| 4 | 价格带分布 | 按价格区间统计 | 平台数据 |
| 5 | 评分分布 | 按评分区间统计 | 平台数据 |
| 6 | 头部品牌TOP20 | 按门店数排序 | 平台数据 |
| 7 | 细分品类占比 | 按品类分类统计 | 关键词匹配 |
| 8 | 市场集中度CR5/CR10 | TOP品牌占比 | 计算得出 |

**额外补充指标**（v3.0新增）：
- 各区平均价格
- 各区平均评分
- 价格-评分交叉分析
- 闭店率估算（高德无法获取，用行业经验补充说明）

## 三、报告生成硬规则

### 规则4：一页纸决策结论（第1页）
**第1页必须是「核心决策结论」**，包含7个核心结论：

```
1. 综合进入评级：□推荐 □谨慎 □不建议
2. 市场饱和度：□蓝海 □适中 □红海
3. 建议切入价格带：___元 - ___元
4. 优先关注行政区：3个
5. 核心机会点：3条
6. 核心风险点：3条
7. 下一步动作建议
```

### 规则5：每个结论带数据来源
**禁止"凭感觉"、"一般认为"**。每条结论必须带：
- 数据来源（如"高德POI 3013家"）
- 量化依据（如"占比43.5%"）
- 对比参照（如"高于全市均值"）

### 规则6：竞品榜单分维度
**禁止单维度榜单**。必须≥4个维度：

| 榜单维度 | 用途 |
|----------|------|
| 门店数榜 | 看规模 |
| 评价数榜 | 看人气 |
| 存活时间榜 | 看老牌（去掉旅游+网红） |
| 区域分布榜 | 看势力范围 |

### 规则7：消费力+成本量化
**禁止只列价格**。必须包含：

| 维度 | 必含数据 |
|------|----------|
| 各区房价 | 元/㎡ |
| 各区人均餐饮消费 | 元/月 |
| 各区客群年龄结构 | 主力年龄段 |
| 商圈租金区间 | 元/㎡/月 |
| 转让费行情 | 万元 |
| 商场扣点 | 百分比 |

### 规则8：Checklist带量化标准
**禁止"临街旺铺"这种空话**。每条必须带量化标准：

```
❌ 错误：周边3km内无同类强势竞品
✅ 正确：周边1.5km内无客单价±20%的同类门店
```

```
附录：数据来源与局限性说明
```

## 七、失败案例（必须避免）

**v1.0的7个硬伤**（用户亲述，逐条避免）：

| # | 硬伤描述 | 用户原话 | 避免方法 |
|---|----------|----------|----------|
| 1 | 定位模糊（既不像城市级也不像商圈级） | "卡在中间，价值感不强" | 报告开头写明"本报告定位=城市级" |
| 2 | 结论主观断言，缺数据支撑 | "望京竞品相对少，但实际是红海" | 每条结论必须带"数据依据" |
| 3 | 城市级核心指标全部缺失 | "缺少全市总量/饱和度/密度排名/CR5" | 强制8项指标清单，缺一不可交付 |
| 4 | 竞品榜单单维度 | "只按评价数排名，没有区分门店属性" | 4维度榜单：门店数/评价数/存活时间/区域 |
| 5 | 消费力+成本维度缺失 | "不知道成本就没法算回本周期" | 必须有算账表：投入/营收/回本周 |
| 6 | CheckList太泛 | "什么叫主动线？没有量化标准" | 每条带量化标准（如"周边1.5km内无同类"） |
| 7 | 缺一页纸决策结论 | "用户要自己提炼，体验感打折扣" | 第1页必须是7要素结论表 |

**v3.0 Review新增专业硬伤（85分版必须规避）**：

| 级别 | 硬伤 | 规避方法 |
|------|------|----------|
| P0-1 | 回本测算公式错误（200×0.7=餐位数） | 餐位数=实用面积÷2㎡/位；月营收=餐位×客单×翻台×30天 |
| P0-2 | 数据口径前后不统一 | 明确"后续分析统一采用XX口径，不同源不做跨口径对比" |
| P1-1 | 头部品牌缺品类边界说明 | 加注"广义热烹品类，含XX；狭义火锅应剔除XX" |
| P1-2 | 区域密度缺建成区说明 | 加注"全域密度≠建成区密度，建成区约为整体的5-8倍" |
| P2-1 | 缺"中端死亡带"行业定论 | 90-130元区间是"尴尬死亡带"（综合火锅），但潮汕牛肉例外有机会 |
| P2-2 | 缺评分爬分规则 | 新店前3个月评价权重最高，1条差评影响是老店3倍 |
| P2-3 | 缺隐性成本提示 | 实际需预留总投资×10-15%隐性成本（证照/排烟/消防） |
| P2-4 | 缺阴阳街效应提醒 | 同商圈客流差异可达30-50%，需实地踏勘 |
| P2-5 | 升单引导未前置 | 第9章结尾必须加升单提示 |
| P2-6 | 缺一句话结论 | 第1页最开头加粗一句话，3秒get核心观点 |

**v3.0修正后达标**：8/8项指标 ✅
